# 021. Hole Swallow - 블랙홀 아케이드

## 게임 컨셉
**"도시를 삼켜라!"**

플레이어가 조종하는 블랙홀이 작은 물체부터 시작해 점점 큰 물체를 삼키며 성장하는 io 스타일 아케이드 게임. Hole.io의 핵심 메카닉을 HTML5로 구현.

## 핵심 메카닉

### 1. 크기 성장 시스템
- 작은 물체(쓰레기통, 표지판) → 중형(자동차, 나무) → 대형(건물, 트럭)
- 현재 홀 크기보다 작은 물체만 흡수 가능
- 흡수할 때마다 홀 크기 증가

### 2. 탑다운 2D 시점
- 간단한 도시/공원 맵
- 터치/마우스 드래그로 홀 이동
- 물체에 닿으면 자동 흡수

### 3. 시간 제한 모드
- 60초 타이머
- 최대한 많이 삼켜서 점수 획득
- 물체 크기별 점수 차등

## 타겟 유저
- 캐주얼 게이머
- 짧은 세션 선호 (1-2분)
- 단순하지만 중독성 있는 게임 선호
- 전연령대

## 차별점
1. **심플한 2D 구현** - 3D 없이도 핵심 재미 전달
2. **다양한 테마 맵** - 도시, 공원, 해변 등
3. **성취감** - 작은 것부터 건물까지 삼키는 성장감
4. **즉각적 피드백** - 삼킬 때마다 시각/청각 효과

## 수익 모델
- **광고 기반** (세션 종료 후 리워드 광고)
- **스킨 판매** (홀 커스터마이징)
- 현재는 프로토타입으로 광고 없이 순수 게임플레이

## 기술 스택
- **엔진:** Vanilla JavaScript + Canvas API
- **렌더링:** 2D Canvas (탑다운 뷰)
- **물리:** 간단한 거리 계산 (충돌 = 원형 겹침)
- **배포:** GitHub Pages

## 상세 설계

### 시스템 설계

```
[Game State]
├── hole: { x, y, radius, score }
├── objects: [ { x, y, size, type, points } ]
├── timer: 60
└── gameOver: boolean

[Game Loop]
1. 입력 처리 (터치/마우스 → 홀 이동)
2. 충돌 검사 (홀과 물체)
3. 흡수 처리 (홀 성장)
4. 타이머 갱신
5. 렌더링
```

### 물체 타입 & 크기
| 타입 | 크기 | 점수 | 필요 홀 크기 |
|------|------|------|-------------|
| 쓰레기통 | 15 | 10 | 20+ |
| 벤치 | 25 | 20 | 30+ |
| 자동차 | 40 | 50 | 45+ |
| 나무 | 50 | 70 | 55+ |
| 트럭 | 60 | 100 | 65+ |
| 작은 건물 | 80 | 200 | 85+ |
| 큰 건물 | 100 | 500 | 105+ |

### UI/UX 플로우
```
[시작 화면]
  ↓ Play 버튼
[게임 플레이]
  - 상단: 타이머 (60s)
  - 상단 우측: 점수
  - 하단: 홀 크기 표시
  ↓ 시간 종료
[결과 화면]
  - 최종 점수
  - 삼킨 물체 수
  - Retry 버튼
```

### 밸런스 파라미터
```javascript
const CONFIG = {
  GAME_DURATION: 60,        // 초
  INITIAL_HOLE_RADIUS: 20,  // 시작 홀 크기
  GROWTH_RATE: 0.5,         // 물체 크기의 0.5배만큼 성장
  HOLE_SPEED: 5,            // 이동 속도
  MAP_WIDTH: 800,
  MAP_HEIGHT: 600,
  OBJECT_COUNT: 50          // 맵 내 물체 수
};
```

### 성장 공식
```
새 홀 크기 = 현재 크기 + (물체 크기 * GROWTH_RATE)
```

### 렌더링 순서
1. 배경 (잔디/도로 패턴)
2. 물체들 (크기순 정렬)
3. 홀 (그라데이션 원)
4. UI (타이머, 점수)
