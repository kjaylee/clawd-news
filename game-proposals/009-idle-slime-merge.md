# 009: Idle Slime Merge (아이들 슬라임 머지)

## 게임 컨셉
슬라임을 합체시켜 더 강력한 슬라임을 만들고, 자동으로 골드를 생산하는 아이들+머지 하이브리드 게임.
탭해서 새 슬라임을 소환하고, 같은 등급끼리 드래그 머지하면 상위 등급으로 진화.
방치해도 골드가 쌓이고, 접속하면 머지로 효율을 극대화.

## 핵심 메카닉
1. **머지 시스템**: 같은 등급 슬라임 2개를 드래그&드롭으로 합체 → 상위 등급 생성
2. **아이들 생산**: 각 슬라임이 초당 골드를 자동 생산 (등급별 생산량 다름)
3. **소환**: 골드로 새 슬라임 소환 (기본 Lv.1)
4. **프레스티지**: 일정 등급 달성 시 리셋 + 영구 보너스 획득
5. **그리드 관리**: 제한된 슬롯(5x4 = 20칸)에서 효율적 배치

## 타겟 유저
- 캐주얼 게이머 (방치형 선호)
- 머지 게임 팬
- 10대~30대, 남녀 불문
- 짧은 세션(1-3분) + 장시간 방치 플레이

## 차별점
- 머지 + 아이들 결합 (두 트렌드 동시 공략)
- 슬라임 캐릭터 귀여움 → 수집 욕구 자극
- 프레스티지 시스템 → 장기 리텐션
- HTML5 → 브라우저 즉시 플레이

## 수익 모델
- 광고: 2배 속도 부스트 (리워드 광고)
- IAP: 프리미엄 슬라임 스킨, 슬롯 확장
- 오프라인 수익 2배 (광고 시청)

## 기술 스택
- HTML5 Canvas + Vanilla JS
- 터치/마우스 드래그&드롭
- localStorage 저장
- 반응형 (모바일 우선)

## 시스템 설계

### 게임 루프
```
1. 아이들 루프 (1초마다):
   - 모든 슬라임의 초당 골드 합산
   - 골드 += 합산값 × (1 + 프레스티지 보너스)
   - UI 업데이트

2. 플레이어 액션:
   - [소환] 버튼 → 골드 차감, 랜덤 빈 칸에 Lv.1 슬라임 생성
   - [드래그] 슬라임 A → 슬라임 B (같은 Lv) → 머지
   - [프레스티지] 버튼 → 전체 리셋, 영구 보너스 +10%
```

### 머지 로직
```
if (slimeA.level === slimeB.level && slimeA.level < MAX_LEVEL) {
  remove(slimeA);
  slimeB.level += 1;
  slimeB.updateAppearance();
  playMergeAnimation();
}
```

### 머지 애니메이션
- 드래그 시: 반투명 슬라임이 손가락 따라감
- 머지 성공: 파티클 이펙트 + 스케일 업 애니메이션
- 새 등급: 짧은 바운스 + 글로우

## 밸런스 파라미터
- 소환 비용: 50G (고정)
- 프레스티지 조건: Lv.7 이상 슬라임 보유
- 프레스티지 보너스: +10%/회
- 오프라인 수익: 실시간의 50%
- 골드 생산 곡선: 지수적 증가 (레벨당 ~2.5배)

## 예상 세션 패턴
1. **첫 세션 (5분)**: Lv.1~3까지 머지 체험, 기본 루프 이해
2. **1일차**: Lv.5~6 도달, 소환 루프 반복
3. **3일차**: 첫 프레스티지, 가속 체감
4. **1주차**: Lv.8~9 도달, 수집 욕구로 리텐션
