## Part 1: 게임 디자인 기초 이론

### 1.1 MDA 프레임워크 (Mechanics → Dynamics → Aesthetics)

**원 논문:** Hunicke, LeBlanc, Zubek — "MDA: A Formal Approach to Game Design and Game Research"

게임을 분석하고 설계하기 위한 가장 기본적인 프레임워크.

#### 세 가지 계층

| 계층 | 정의 | 예시 |
|------|------|------|
| **Mechanics** (메카닉) | 게임의 규칙, 기본 액션, 알고리즘 | 블록 배치, 점프, 매칭, 슈팅 |
| **Dynamics** (다이나믹) | 메카닉 + 플레이어 입력에서 생기는 런타임 행동 | 경쟁, 협동, 전략적 선택 |
| **Aesthetics** (에스테틱) | 플레이어가 느끼는 감정적 반응 | 재미, 몰입, 성취감, 긴장감 |

#### 핵심 원리
- **디자이너 관점:** Mechanics → Dynamics → Aesthetics (만드는 순서)
- **플레이어 관점:** Aesthetics → Dynamics → Mechanics (경험하는 순서)
- 디자이너는 Mechanics만 직접 제어 가능 → Dynamics와 Aesthetics는 간접적으로만 영향

#### 8가지 에스테틱 유형
1. **Sensation** — 감각적 쾌감 (비주얼, 사운드)
2. **Fantasy** — 상상의 세계
3. **Narrative** — 스토리 몰입
4. **Challenge** — 도전과 극복
5. **Fellowship** — 사회적 소속감
6. **Discovery** — 탐험의 즐거움
7. **Expression** — 자기표현
8. **Submission** — 게임 세계에의 몰입 (패스타임)

#### 실전 적용법
1. 게임 기획 시 먼저 목표 Aesthetics(감정)를 정의
2. 이를 만들어낼 Dynamics를 설계
3. Dynamics를 구현할 Mechanics를 코딩
4. 플레이테스트로 의도한 Aesthetics가 나오는지 검증

#### 주의점
- MDA는 게임 메카닉에 너무 집중한다는 비판 존재
- DDE(Design-Dynamics-Experience) 프레임워크가 대안으로 제시됨
- 경험 중심 디자인이나 게이미피케이션에는 보완 필요

### 1.2 핵심 루프 (Core Loop) 설계

**정의:** 플레이어가 반복적으로 수행하는 행동의 순환 구조. 게임 경험의 뼈대.

#### 핵심 루프의 시간 스케일

| 스케일 | 설명 | 예시 (슈터) |
|--------|------|-------------|
| **순간** (Moment) | 1-5초 단위 | 조준→사격→회피 |
| **분** (Minute) | 1-5분 단위 | 퀘스트 수령→완료→보상 |
| **시간** (Hour) | 30-60분 단위 | 장비 강화→새 지역 진출 |
| **일** (Day) | 일일 단위 | 데일리 퀘스트→리워드→다음 목표 |

#### 강한 핵심 루프의 4요소 (Player Centric Framework)
1. **Clarity (명확성)** — 뭘 해야 하는지 즉시 이해
2. **Motivation (동기)** — 왜 해야 하는지 알기
3. **Response (반응)** — 도전에 대한 즉각 피드백
4. **Satisfaction (만족감)** — 노력에 대한 보상감

#### 모바일 게임 핵심 루프 구조
```
[플레이] → [결과] → [보상] → [강화/진행] → [더 어려운 플레이]
                                                    ↓
                                              [메타게임]
```

- **Short Loop:** 1판 플레이 (2-5분)
- **Medium Loop:** 레벨/챕터 진행 (1세션)
- **Long Loop:** 메타게임 진행 (컬렉션, 업그레이드, 시즌)

#### 실전 적용법
- 핵심 루프를 한 문장으로 설명할 수 있어야 함
- "플레이어는 [행동]을 해서 [결과]를 얻고, [보상]으로 [성장]한다"
- 루프가 자연스럽게 반복을 유도해야 ("한 판만 더")

### 1.3 플로우 이론 (Csikszentmihalyi) — 난이도 곡선

**정의:** 미하이 칙센트미하이의 플로우 이론. 도전(Challenge)과 능력(Skill)의 균형이 맞을 때 '몰입(Flow)' 상태에 진입.

#### 플로우 채널 다이어그램
```
높음 │  불안(Anxiety)  │   각성(Arousal)    │
     │                │                    │
도전  │                │   ★ FLOW ★        │
     │                │                    │
     │  무관심(Apathy) │   지루함(Boredom)   │
낮음 │                │                    │
     └────────────────┴────────────────────┘
        낮음          능력(Skill)         높음
```

#### 게임 디자인 적용
- **난이도 너무 높음** → 좌절, 이탈 (불안 영역)
- **난이도 너무 낮음** → 지루함, 이탈 (지루함 영역)
- **적절한 균형** → 몰입, 중독 (플로우 영역)

#### 이상적 난이도 곡선: "톱니 모양" (Sawtooth)
```
난이도
  │    /\    /\    /\
  │   /  \  /  \  /  \
  │  /    \/    \/    \
  │ /                   \
  └─────────────────────→ 시간
     쉬움  중간  어려움  휴식
```

- 점진적 상승 + 주기적 이완(Easy Level)
- 어려운 레벨 후 쉬운 레벨 → 성취감 + 자신감 회복
- 보스 스테이지 전후로 난이도 조절

### 1.4 보상 심리학 (Variable Ratio Reinforcement)

**기반:** B.F. Skinner의 조작적 조건형성(Operant Conditioning)

#### 강화 스케줄 4가지

| 스케줄 | 정의 | 게임 예시 | 효과 |
|--------|------|-----------|------|
| **Fixed Ratio (FR)** | 정해진 횟수 후 보상 | 적 5마리 = 보상 | 예측 가능, 안정적 |
| **Variable Ratio (VR)** | 불규칙 횟수 후 보상 | 가챠, 랜덤 드롭 | **가장 중독적** |
| **Fixed Interval (FI)** | 정해진 시간 후 보상 | 데일리 보상 | 정기적 접속 유도 |
| **Variable Interval (VI)** | 불규칙 시간 후 보상 | 랜덤 이벤트 팝업 | 지속적 관심 유지 |

#### Variable Ratio가 가장 강력한 이유
- 예측 불가능성 → "다음엔 나올지도" 기대감
- 높은 반응률 + 소거 저항(멈추기 어려움)
- 슬롯머신, 가챠, 랜덤 드롭이 모두 VR 스케줄
- 도파민 시스템 직접 자극

#### 윤리적 게임 디자인
- VR 스케줄은 강력하지만 남용 시 플레이어 착취
- "게임이 재미있어서 하는 것" vs "멈출 수 없어서 하는 것" 구분 필요
- 캐주얼 게임: VR을 보조적으로 사용 (메인 보상은 FR/FI)
- 핵심: **보상의 예측 가능성과 불확실성의 적절한 배합**

#### 실전 적용법
- **메인 보상:** FR — 레벨 클리어 시 확정 보상 (공정함)
- **서브 보상:** VR — 랜덤 보너스 드롭 (흥미 유지)
- **데일리 보상:** FI — 매일 접속 보상 (루틴화)
- **이벤트 보상:** VI — 갑작스러운 이벤트 (놀라움)

### 1.5 플레이어 유형 (Bartle Taxonomy)

**출처:** Richard Bartle, 1996 — 멀티플레이어 게임 플레이어 분류

#### 4가지 플레이어 유형

| 유형 | 동기 | 행동 | 게임 요소 |
|------|------|------|-----------|
| **Achiever** (성취형) | 목표 달성 | 포인트, 레벨, 도전 클리어 | 리더보드, 배지, 진행률 |
| **Explorer** (탐험형) | 발견과 학습 | 맵 탐험, 히든 요소 찾기 | 비밀 방, 이스터에그, 로어 |
| **Socializer** (소셜형) | 타인과의 교류 | 채팅, 협동, 커뮤니티 | 길드, 채팅, 선물 |
| **Killer** (경쟁형) | 타인에 대한 우위 | PvP, 랭킹 경쟁, 방해 | PvP, 랭킹, 대결 |

#### 2축 모델
```
           Acting on
               │
    Killers    │    Achievers
               │
  Players ─────┼───── World
               │
    Socializers│    Explorers
               │
           Interacting with
```

#### 모바일/캐주얼 게임 적용
- 대부분의 캐주얼 플레이어: **Achiever + Socializer** 경향
- 싱글플레이 모바일: **Achiever** 중심 설계 (점수, 진행, 컬렉션)
- 소셜 기능 추가 시: **Socializer** 요소 (리더보드, 공유, 협동)
- 하드코어 요소: **Killer** (PvP), **Explorer** (히든 콘텐츠)

## Part 2: 모바일/캐주얼 게임 기획

### 2.1 온보딩 설계 (FTUE - First Time User Experience)

**핵심 원칙:** 첫 인상은 한 번뿐. 첫 세션에서 플레이어를 사로잡지 못하면 두 번째 기회는 없다.

#### FTUE 황금 규칙 7가지

1. **10초 내 게임 시작** — 계정 생성, IAP 팝업, 긴 로딩 절대 금지
2. **행동으로 가르치기** — 텍스트 벽 대신 "플레이하면서 배우기"
3. **Level 0** — 실패 불가능한 첫 레벨로 기본 조작 학습
4. **가치 즉시 보여주기** — 게임의 가장 매력적인 점을 첫 1분에
5. **스킵 가능** — 모든 인트로/컷신은 스킵 가능해야
6. **보상으로 동기부여** — 작은 마일스톤마다 축하 + 보상
7. **미래를 보여주기** — 앞으로 잠금해제할 캐릭터/아이템 미리보기

#### FTUE 안티패턴 (하지 말 것)
- ❌ 강제 계정 생성
- ❌ 긴 텍스트 튜토리얼
- ❌ 첫 세션에 IAP/광고 팝업
- ❌ 한번에 모든 기능 설명
- ❌ 스킵 불가능한 인트로
- ❌ 복잡한 UI 한꺼번에 노출

#### 퍼널 (측정 지표)
```
다운로드 → 첫 실행 → 튜토리얼 시작 → 튜토리얼 완료 → 첫 레벨 클리어 → 두번째 세션
  100%      85%         75%            60%           50%           30%
```
- 각 단계의 이탈률(Drop-off) 추적
- 튜토리얼 완료율 60% 미만이면 설계 재검토

### 2.2 세션 설계 (1회 플레이 2-5분)

#### 모바일 세션의 특성
- 평균 세션 길이: 캐주얼 3-7분, 미드코어 10-15분
- 플레이 맥락: 이동 중, 대기 중, 짧은 쉬는 시간
- **세션이 짧아야 하는 이유:** 습관 형성 → 장기 리텐션

#### 세션 설계 원칙
1. **빠른 시작**: 앱 열고 3초 내 플레이 가능
2. **자연스러운 끝점**: 레벨 클리어, 라이프 소진 등
3. **중간 이탈 허용**: 언제든 나갔다 돌아와도 OK
4. **"한 판만 더" 유도**: 세션 끝에 다음 도전 미리보기
5. **세션 간 진행 유지**: 저장, 오프라인 진행, 보상 축적

#### 시간별 세션 구조
```
0:00 - 0:05  → 앱 시작, 즉시 게임 진입
0:05 - 0:30  → 게임 준비 (레벨 선택/로딩)
0:30 - 3:00  → 핵심 게임플레이
3:00 - 3:30  → 결과 화면, 보상, 피드백
3:30 - 4:00  → "다음 레벨?" 또는 "리트라이?" 프롬프트
4:00 - 5:00  → 메타게임 확인 (업그레이드, 인벤토리)
```

### 2.3 리텐션 메카닉

#### Day-1, Day-7, Day-30 리텐션 벤치마크 (모바일 퍼즐)
| 기간 | 업계 평균 | 상위 25% | Block Blast |
|------|-----------|----------|-------------|
| D1 | ~28% | ~40% | 41.8% |
| D7 | ~8% | ~15% | 26.9% |
| D30 | ~3% | ~6% | 13.3% |

#### 리텐션 메카닉 도구

**1. 데일리 보상 (Daily Rewards)**
- 7일 로그인 보상 (에스컬레이팅): 1일차 100코인 → 7일차 1000코인
- 연속 보너스: 연속 로그인 시 보상 배수 증가
- 위험: 놓치면 리셋 → 스트레스 (대안: 부분 리셋)

**2. 스트릭 (Streak)**
- 연속 활동 카운터 (Duolingo 모델)
- 깨지면 아쉬움 → "오늘도 해야지" 동기
- 프리즈/복구 아이템 (유료 가능)

**3. 챌린지/퀘스트**
- 데일리 챌린지: "오늘의 미션 3개 완수"
- 위클리 챌린지: 더 큰 보상
- 시즌 패스: 장기 목표 + 단계별 보상

**4. 제한된 에너지 (Energy System)**
- 플레이 횟수 제한 → 시간 경과 or 결제로 충전
- 장점: 세션 조절, 수익화 연결
- 단점: 리텐션에 역효과 가능 (Block Blast는 의도적으로 에너지 없음)

**5. 소셜 리텐션**
- 친구 리더보드
- 라이프/선물 보내기
- 길드/팀 목표

### 2.4 난이도 밸런싱 (동적 난이도 조절 DDA)

**정의:** 플레이어의 실력에 맞춰 실시간으로 난이도를 조절하는 시스템.

#### DDA 구현 방법

**1. 성과 기반 조절**
- 연속 클리어 → 난이도 ↑
- 연속 실패 → 난이도 ↓
- 매개변수: 적 수, 시간 제한, 보조 아이템 빈도

**2. 은밀한 조절 (Stealth DDA)**
- 플레이어가 눈치채지 못하게
- 예: 충돌 판정 약간 관대하게, 힌트 자동 제공
- Block Blast: "게임이 플레이어 행동을 예측하고 난이도를 조절"

**3. 선택적 난이도**
- 플레이어가 직접 선택 (Easy/Normal/Hard)
- 또는: 리트라이 시 자동 힌트 제공

#### DDA 설계 주의사항
- 조절이 너무 눈에 띄면 "게임이 나를 봐준다" → 성취감 ↓
- 너무 미묘하면 효과 없음
- 핵심: **플레이어가 "내가 잘해서 클리어했다"고 느끼게**

### 2.5 메타게임 설계

**정의:** 핵심 게임플레이 바깥의 진행 시스템. "게임 안의 게임."

#### 메타게임 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **진행 시스템** | 레벨/챕터/맵 | Candy Crush의 맵 진행 |
| **컬렉션** | 캐릭터/아이템 수집 | 포켓몬, 카드 수집 |
| **업그레이드** | 능력치 강화 | 무기 강화, 스킬 트리 |
| **커스터마이징** | 외형 변경 | 스킨, 테마 |
| **소셜** | 길드, 랭킹 | 클랜 전쟁, 리더보드 |
| **시즌** | 한정 이벤트/컨텐츠 | 배틀패스, 시즌 랭킹 |

#### 메타게임 설계 원칙
1. **핵심 루프와 연결**: 메타 진행이 핵심 플레이를 더 재미있게
2. **장기 목표 제공**: "이것을 모으면..." "여기까지 가면..."
3. **점진적 해금**: 처음엔 단순 → 점점 기능 추가
4. **복수의 진행축**: 한 가지가 막혀도 다른 곳에서 진행 가능

## Part 3: 수익화 기획

### 3.1 광고 통합 설계

#### 광고 유형별 특성

| 유형 | 타이밍 | 플레이어 경험 | 수익 |
|------|--------|--------------|------|
| **배너** | 상시 표시 | 방해 적음, 무시됨 | 낮음 (CPM $0.5-2) |
| **인터스티셜** | 세션 전환 시 | 중간 방해 | 중간 (CPM $3-10) |
| **리워디드 비디오** | 플레이어 선택 | 최소 방해 | 높음 (CPM $10-30) |

#### 광고 타이밍 황금 규칙
1. **인터스티셜**: 레벨 사이, 게임오버 후 (게임플레이 중 절대 금지)
2. **리워디드**: "한 번 더 하시겠어요?", "보상 2배", "힌트 받기"
3. **배너**: 메뉴 화면, 결과 화면 (게임플레이 화면은 피하기)
4. **빈도 제한**: 인터스티셜 3분 간격 이상
5. **첫 세션 광고 최소화**: 새 유저는 광고로 이탈 가능성 높음

#### Block Blast 사례
- 배너: 플레이 중 하단 상시
- 인터스티셜: 레벨 종료 후, 게임오버 후
- 리워디드: 리트라이 기회
- IAP: 없음 (순수 광고 수익 $584K/일)
- 핵심: 에너지 시스템 없음 → 무한 플레이 → 광고 노출 극대화

### 3.2 IAP 설계 원칙

#### IAP 유형

| 유형 | 설명 | 예시 | 윤리성 |
|------|------|------|--------|
| **Pay-to-Skip** | 시간/노력 건너뛰기 | 에너지 충전, 즉시 완료 | ⚠️ 밸런스 주의 |
| **Cosmetic** | 외형만 변경 | 스킨, 테마, 이펙트 | ✅ 가장 공정 |
| **Energy** | 플레이 횟수 구매 | 라이프 충전 | ⚠️ 게임성 영향 |
| **Power** | 게임 내 이점 | 부스터, 파워업 | ❌ Pay-to-Win 우려 |
| **Content** | 추가 콘텐츠 | 레벨 팩, 캐릭터 | ✅ 가치 교환 명확 |
| **No-Ads** | 광고 제거 | 일회성 구매 | ✅ Win-Win |

#### IAP 설계 원칙
1. **무과금 경험 보장**: 돈 안 써도 게임 끝까지 플레이 가능
2. **가치 명확**: 뭘 사는지, 왜 좋은지 바로 이해
3. **첫 결제 유도**: 스타터 팩 (90% 할인 등)
4. **가격 앵커링**: 다양한 가격대 (₩1,000 ~ ₩100,000)
5. **희소성/긴급성**: 한정 판매, 타임 딜 (과도하면 역효과)

#### Stars 연동 기획 포인트

3. **결제 플로우**: Bot API의 `sendInvoice` / WebApp의 `openInvoice`
4. **서버 검증**: 결제 완료 후 서버에서 확인 후 상품 지급

## Part 4: 레벨 디자인

### 4.1 퍼즐 게임 레벨 디자인 원칙

#### 기본 원칙
1. **새 메카닉 도입 → 연습 → 조합**: 한 번에 하나씩
2. **실패해도 배움이 있어야**: "왜 실패했는지" 명확
3. **풀이가 보여야**: 해법의 실마리 제공 (힌트 아님, 단서)
4. **복수 해법 허용**: 하나의 정답만 있으면 좌절 유발
5. **레벨 길이 적절**: 3분 이상 걸리면 모바일에선 이탈

#### 레벨 구조 (10레벨 기준)
```
레벨 1-2: 기본 메카닉 소개 (실패 불가)
레벨 3-4: 기본 메카닉 응용
레벨 5:   새 메카닉 도입
레벨 6-7: 새 메카닉 연습
레벨 8-9: 두 메카닉 조합
레벨 10:  도전 레벨 (보스급)
```

### 4.2 프로시저럴 생성 vs 핸드크래프트

| 특성 | 프로시저럴 | 핸드크래프트 |
|------|-----------|-------------|
| **콘텐츠 양** | 무한 | 제한적 |
| **품질 보장** | 어려움 | 높음 |
| **개발 비용** | 초기 높음 (알고리즘) | 선형 증가 |
| **리플레이** | 매번 다름 | 동일 |
| **적합 장르** | 로그라이크, 무한모드 | 스토리 퍼즐, 캠페인 |

#### 하이브리드 접근 (권장)
- **캠페인/스토리**: 핸드크래프트 레벨
- **무한/일일 모드**: 프로시저럴 생성
- **검증**: 자동 풀이 검증기로 생성된 레벨 필터링

### 4.3 난이도 S-커브

```
난이도
 100% │                          ___/---
      │                      __/
      │                   __/
      │                 _/
  50% │              __/
      │            _/
      │          _/
      │        _/
      │      _/
   0% │___--/
      └──────────────────────────→ 레벨 수
        초반(완만) 중반(가파름) 후반(완만)
```

- **초반 (레벨 1-20%)**: 완만한 상승 → 학습 + 자신감 형성
- **중반 (레벨 20-80%)**: 가파른 상승 → 핵심 도전 구간
- **후반 (레벨 80-100%)**: 다시 완만 → 마스터리 느낌
- **핵심**: 절대 수직 절벽 없이 → 항상 "조금만 더 하면"

---

### 4.4 풀이 보장 알고리즘

#### 접근법 3가지

**1. 역방향 생성 (Backward Generation)**
- 완성된 상태에서 시작 → 역으로 해체
- 퍼즐 풀이 경로가 보장됨
- 예: 스도쿠, 지뢰찾기

**2. 시뮬레이션 기반 검증**
- 레벨 생성 후 AI/솔버가 풀어봄
- 풀리지 않으면 폐기하고 재생성
- 예: Match-3 레벨

**3. 제약 기반 생성 (Constraint-based)**
- Wave Function Collapse 등 제약 조건을 만족하는 생성
- 구조적으로 풀이 보장
- 예: 절차적 던전 생성

#### 실전 주의사항
- 풀이 가능 ≠ 재미있음 (난이도 적절성 별도 검증)
- "최소 N수 안에 풀이 가능" 조건 추가
- 막힘 방지: 셔플/힌트 시스템 필수

## Part 5: 성공한 게임 해부

### 5.1 Block Blast — 무엇이 중독성을 만드는가

> 222M+ DL (2024), 40M+ DAU, $17.5M/월 수익

#### 핵심 데이터
| 지표 | Block Blast | 업계 평균 | 차이 |
|------|-------------|-----------|------|
| D1 리텐션 | 41.8% | 28% | +46% |
| D7 리텐션 | 26.9% | 8% | +225% |
| D30 리텐션 | 13.3% | 3% | +333% |

#### 성공 요인 분석

**1. 극단적 단순성**
- 3개 블록을 8×8 보드에 배치 → 줄 완성 시 제거
- 규칙 설명 10초 가능
- 테트리스 영감이지만 속도 대신 논리

**2. 무한 플레이 (No Energy)**
- 에너지/라이프 시스템 없음
- 오프라인 플레이 가능
- "멈추는 이유"가 게임 밖에만 존재

**3. 지속적 피드백**
- "Unbelievable!", "Perfect!" 등 격려 메시지
- 사운드 이펙트 + 진동
- 도파민 트리거: 줄 완성의 시각적 만족감

**4. DDA (Dynamic Difficulty Adjustment)**
- 플레이어 행동 예측 → 난이도 자동 조절
- "어렵지만 불가능하지 않은" 상태 유지
- A/B 테스팅으로 지속 최적화

**5. 두 가지 모드**
- Classic: 하이스코어 도전 (무한)
- Adventure: 96레벨 (주간 갱신)
- 다른 동기의 플레이어 모두 포섭

**6. 순수 광고 수익 모델**
- IAP 없음, 광고만으로 $584K/일
- 리워디드 광고: 리트라이 기회
- 결과: 플레이어 지갑 부담 없음 → 리텐션 ↑

#### 교훈
- **단순함이 최고의 중독성** — 복잡한 메카닉 필요 없음
- **에너지 시스템 제거 → 리텐션 폭발** (광고 모델과 시너지)
- **DDA는 필수** — 플레이어 맞춤 경험
- **소셜 기능 없이도 바이럴 가능** — 게임 자체의 힘

### 5.2 2048 — 심플함의 힘

> 2014년 출시, 전 세계 바이럴, 수백 개의 클론 탄생

#### 성공 요인

**1. 한 줄로 설명 가능한 규칙**
- 4×4 그리드에서 같은 숫자 합치기 → 2048 만들기
- 스와이프 한 방향 → 모든 타일 이동
- 3초면 이해, 마스터는 수시간

**2. "배우기 쉽고 마스터 어려운" 이상적 구조**
- 초반: 직관적으로 플레이
- 중반: 전략 패턴 발견 (코너 전략)
- 고수: 최적화, 높은 점수 도전

**3. 완벽한 플로우 상태**
- 클리어 보장 아님 → 적절한 긴장감
- 큰 숫자 합칠 때 성취감
- 실패해도 "다음엔 더 잘 할 수 있다" 느낌

**4. 무한 리플레이 + 짧은 세션**
- 1판 2-5분
- 오프라인 가능
- 상태 저장

**5. 오픈소스 + 심플 구현**
- HTML5 단일 파일
- 수백 개의 변형 탄생 → 커뮤니티 효과
- 텔레그램 Mini App에 이상적

#### 교훈
- **규칙의 단순성이 중독성의 핵심**
- **깊이는 복잡한 규칙이 아니라 창발적 전략에서**
- **오픈소스/클론 가능성 = 바이럴의 촉매**

### 5.3 Vampire Survivors — 뱀서 장르의 핵심

> 2022년 출시, $2.69, 인디 메가히트, "Bullet Heaven" 장르 창시

#### 핵심 설계

**1. 극도로 낮은 조작 문턱**
- 이동만 조작 (방향키/스틱)
- 공격은 100% 자동
- "아무것도 안 해도 뭔가 일어나는" 즐거움
- → 모바일/캐주얼 전환 용이

**2. 파워 판타지 (Power Fantasy)**
- 시작: 약한 캐릭터, 하나의 무기
- 30분 후: 화면 가득 채우는 무기 이펙트
- 핵심: "나는 이제 신이다" 느낌
- Bullet Hell의 반대 = Bullet Heaven

**3. 리워드 빈도의 최적화**
- 약 23초마다 레벨업 → 보상 선택
- 일반 게임: 2분 30초에 1번 보상 (12개 아이템/30분)
- VS: 80회 보상/30분 (8티어 × 6무기 + 5티어 × 6패시브)
- 잦은 보상 = 잦은 도파민 → 중독

**4. 로그라이크 진행**
- 런(Run) 실패해도 영구 해금 (캐릭터, 무기, 스테이지)
- "실패해도 진행된다" → 좌절감 최소
- 메타 진행이 다음 런 동기부여

**5. 시각적 과부하의 재미**
- 수백 마리의 적, 수십 개의 이펙트
- 카오스 자체가 즐거움
- "뇌를 끄고 즐기는" 경험

#### 교훈
- **조작 단순화가 접근성의 핵심** (이동만!)
- **보상 빈도를 극대화** → 23초마다 선택
- **파워 판타지는 보편적 재미** → 장르 불문
- **실패에도 진행** → 로그라이크의 핵심 리텐션

### 5.4 Candy Crush — Match-3의 교과서

> 2012년 출시, 역대 최고 수익 모바일 게임 중 하나

#### 핵심 설계

**1. Match-3의 정석**
- 같은 색 3개 이상 매칭 → 제거
- 특수 캔디 조합 → 강력한 효과
- 직관적이지만 전략적 깊이

**2. 에너지 시스템 + 소셜**
- 라이프 5개 → 소진 시 기다림 or 결제
- 친구에게 라이프 요청 → 소셜 바이럴
- Facebook 연동 → 친구 진행 비교

**3. 난이도 절벽 = 전환점**
- 의도적으로 어려운 레벨 배치
- "거의 클리어" 경험 → 도파민 + 좌절
- 해결책 제시: 부스터 구매 (IAP)
- Variable Ratio: "이번엔 될 수도..."

**4. 수익화 마스터클래스**
- "Wait or Pay" 메카닉
- 부스터/파워업 판매
- 골드바 (프리미엄 화폐)
- Charm (난이도 감소 아이템)
- 일회성 구매 아닌 반복 소비

**5. 맵 진행의 심리학**
- 수백 개의 레벨 → "끝이 보이지 않는 여정"
- 맵 비주얼 → 진행 감각
- 새로운 메카닉 주기적 도입

#### 교훈
- **"거의 성공" 경험이 결제를 유도** (Near-Miss Effect)
- **소셜 연결 = 무료 마케팅 + 리텐션**
- **에너지 시스템의 양날의 검** — 수익 ↑ but 리텐션 리스크
- **반복 가능한 소비 모델** (일회성 구매보다 효과적)

---

### 5.5 Among Us — 소셜 메카닉

> 2018년 출시, 2020년 바이럴, 500M+ DL

#### 핵심 설계

**1. 소셜 추론 (Social Deduction)**
- 크루원 vs 임포스터
- 투표로 추방 → 대화/논쟁이 핵심
- "누가 거짓말하는가?" → 사회적 긴장감

**2. 비대칭 플레이**
- 크루원: 미션 수행 + 임포스터 찾기
- 임포스터: 방해 + 은밀한 제거
- 같은 게임, 완전히 다른 경험

**3. 물리적 행동 + 소셜 행동**
- 미션(태스크)이 있어 "왜 거기 있었는지" 설명 필요
- 단순한 마피아 게임과 차별화
- "난 전기실에서 미션 하고 있었어" → 알리바이 시스템

**4. 접근성**
- 무료(광고) + 크로스플랫폼
- 규칙 단순: "임포스터 찾아 투표" or "미션 완수"
- 비게이머도 참여 가능

**5. 스트리밍 친화**
- 관전자도 재미 (누가 임포스터인지 알면서 보기)
- Twitch/YouTube에서 폭발적 성장
- 사회적 현상 → 밈, 커뮤니티

#### 교훈
- **사회적 상호작용이 핵심인 게임은 바이럴에 유리**
- **비대칭 역할 = 리플레이 동기** (두 가지 경험)
- **스트리밍 친화 설계** = 무료 마케팅
- **물리적 + 소셜 행동의 결합** = 깊이

## Part 6: 실패 패턴

### 6.1 게임성 미성립의 원인들

1. **핵심 루프 없음**: 반복할 만한 재미있는 행동이 없음
2. **피드백 부재**: 행동 → 결과 연결이 불명확
3. **목표 불명확**: 뭘 해야 하는지 모름
4. **보상 감 없음**: 노력에 대한 보상이 약하거나 지연
5. **창발성 부재**: 같은 행동의 반복만 → 지루함
6. **밸런스 붕괴**: 너무 쉽거나 너무 어려움
7. **기술적 결함**: 버그, 프레임 드롭 → 즐거움 파괴

### 6.2 난이도 절벽 (Difficulty Cliff)

```
난이도 │         │
       │     ████│████
       │     █   │    █
       │     █   │    █  ← 이탈 발생!
       │    █    │
       │ ███     │
       │█        │
       └─────────┼────→ 레벨
              문제 레벨
```

**원인:**
- 새 메카닉을 충분한 학습 없이 도입
- 한 번에 여러 메카닉 조합 요구
- 테스터 편향: "나는 할 수 있으니까 괜찮겠지"

**해결:**
- 데이터 기반: 레벨별 클리어율/리텐션 추적
- 80% 이상 클리어율 목표 (퍼즐 장르)
- 어려운 레벨 후 쉬운 레벨 (회복)
- DDA 도입

### 6.3 모바일 UX 실수

| 실수 | 결과 | 해결 |
|------|------|------|
| 작은 터치 영역 (<44px) | 오터치 → 좌절 | 최소 44×44px |
| 느린 피드백 (>100ms) | "고장났나?" 느낌 | 즉각 시각/청각 피드백 |
| 튜토리얼 텍스트 벽 | 이탈 | 행동으로 가르치기 |
| 복잡한 초기 UI | 정보 과부하 | 점진적 해금 |
| 스킵 불가 컷신 | 짜증 | 모든 것 스킵 가능 |
| 긴 로딩 (>3초) | 이탈 | 비동기 로딩, 프리로드 |
| 세로/가로 전환 깨짐 | 불신감 | 고정 방향 or 반응형 |
| 작은 폰트 (<14px) | 읽기 불편 | 최소 14px |
| 중요 버튼 뒤로 숨김 | 못 찾음 | 주요 CTA 항상 표시 |

### 6.4 리텐션 킬러

1. **첫 세션 광고 폭격** → D1 리텐션 파괴
2. **에너지 고갈 + 긴 대기** → "안 하면 안 하지" 심리
3. **불공정한 난이도** → "이건 불가능해" 이탈
4. **업데이트 부재** → 콘텐츠 소비 완료 후 이탈
5. **버그/크래시** → 신뢰 상실
6. **소셜 기능 강제** → 혼자 하고 싶은 사람 이탈
7. **복잡한 경제 시스템** → 뭘 사야 하는지 혼란
8. **진행 막힘** → 한 곳에서 막히면 모든 것 정지
