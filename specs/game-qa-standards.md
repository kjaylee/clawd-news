# 게임 QA 표준 (2026-01-30)

## 자동 QA 봇 테스트 항목

### 1. 기본 로드
- [ ] 페이지 로드 성공 (5초 내)
- [ ] JS 콘솔 에러 없음 (TypeError, ReferenceError 등)

### 2. 시작 플로우
- [ ] 시작 버튼 존재 & 클릭 가능
- [ ] 시작 버튼 클릭 후 게임 화면 전환 확인
- [ ] 시작 직후 즉사/GameOver 안 됨 (최소 5초 생존)

### 3. 터치 인터랙션 (핵심!)
- [ ] 게임 영역 터치 반응 있음 (터치 전후 화면 변화)
- [ ] 버튼 터치 시 실제 동작 수행됨 (UI 변경, 상태 변화)
- [ ] 모든 버튼이 클릭 가능 (disabled 상태가 아닌 한)
- [ ] 터치 영역이 충분히 큼 (최소 44x44px)

### 4. 애니메이션 속도
- [ ] 인간이 인지 가능한 속도 (최소 200ms 이상 지속)
- [ ] 빠른 번쩍임 효과 금지 (50ms 이하 애니메이션 금지)
- [ ] 상태 전환 시 최소 300ms 전환 시간
- [ ] 점수/레벨업 표시가 최소 1초 이상 유지

### 5. 게임 메카닉
- [ ] 승리/패배 조건이 명확
- [ ] 진행(레벨/점수) 시스템 존재
- [ ] 재시작 가능
- [ ] 튜토리얼 또는 직관적 UI

### 6. 모바일 최적화
- [ ] 뷰포트 390x844 정상 표시
- [ ] 스크롤 없이 전체 게임 표시
- [ ] 가로 스크롤 없음
- [ ] safe area 적용 (텔레그램 Mini App)

## FAIL 기준 (하나라도 해당 시)
- 페이지 로드 실패
- 치명적 JS 에러
- 시작 버튼 없음/클릭 불가
- 시작 후 5초 내 즉사
- 터치 무반응
- 빈 화면/깨진 레이아웃
- 게임 메카닉 미성립

## 애니메이션 가이드라인
- transition: 최소 200ms
- 점수/보상 표시: 최소 1000ms
- 화면 전환: 300-500ms
- 파티클/이펙트: 최소 150ms
- **절대 금지**: 50ms 이하 번쩍임, requestAnimationFrame으로 1프레임만 표시
